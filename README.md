# Endstate Matrix — Wuthering Waves Team Manager

A drag-and-drop team composition manager for Wuthering Waves. Build teams, assign elements, organize by role, and save/load your layouts.

## Features

### Adding Characters
- **Bundled characters** are auto-loaded from `characters.json` on page start
- **Upload Icons** button (top toolbar, folder icon) — select a folder of `.png` images. Each file becomes a character in the Unsorted zone using the filename (without extension) as the name. Supports `webkitdirectory` for batch folder upload
- **Spacebar** — opens the search overlay. Type character names and press Enter to instantly create a team with those units. Names autocomplete from loaded icons

### Building Teams
- **Drag units** from the roster (Main DPS, Sub-DPS, Support, Unsorted) onto the workspace to create teams
- **Drop on existing teams** to add a unit (max 3 per team)
- **Drag teams** by their body to reposition. In freeform mode, position is absolute; in rows mode, teams snap into rows
- **x2 charges**: click a unit in the roster to toggle "2 charges" (badge appears). Dragging a x2 unit into a team creates a copy, keeping one copy in the roster

### Elements
- Click the **element badge** (colored circle on a team's top bar) to open the element picker. Single-click applies one element; clicking a second element applies dual-element (50/50 gradient)
- In **Rows layout**, clicking a row's element badge assigns that element to the row. All teams in that row auto-inherit the row's element
- **Drag a team between rows** — the team's element updates to match the target row

### Team Management
- **Lock** — click the lock icon on a team's top bar to lock it (prevents unit rearrangement)
- **Delete** — the × button removes the team and returns all units to Unsorted
- **Rename** — click the team name label to edit inline
- **Lock toggle via drag**: in freeform mode, dragging a team to the right/bottom edge toggles lock state

### Layout Modes
- **Freeform** — absolute-positioned canvas with zoom/pan and snap-to-grid. Teams are placed freely
- **Rows** — six element rows (Aero, Electro, Spectro, Fusion, Glacio, Havoc) plus an unassigned row. Teams flow inside rows

  - **Row Direction**: horizontal or vertical (default, rows as columns, teams flow top-to-bottom)
  - **Row Alignment**: left / center (default) / right alignment for teams inside rows
  - **Drag rows** by their top bar to reorder

### Roster Organization
- **Unsorted / Unowned** zones at the bottom — drag unwanted units into Unowned. Both collapse together by clicking their title bar
- **Main DPS / Sub-DPS / Support** categories — drag units between them to organize by role

### Saving & Loading
- **Save** (floppy disk icon) — downloads a `wuwa-team-manager.json` file containing all teams, roster positions, elements, charges, locked state, and layout settings
- **Load** (file icon) — upload a previously saved `.json` to restore everything. Duplicate units in roster are auto-removed, charges adjusted

### Rover Gender
- Toggle between Male/Female in Settings. Only the active Rover variant appears in the roster; existing Rover icons swap on toggle

### Settings
- **Show Character Names** — toggle unit name labels on/off (default: off)
- **Snap to Grid** — snap teams to 40px grid in freeform mode (default: off)
- **Layout Style** — switch between Freeform and Rows (default: Rows)
- **Row Direction** — horizontal or vertical (default: vertical)
- **Row Alignment** — left / center / right (default: center)
- **Rover Icon** — Male / Female (default: Male)

## Deployment

Static site — no backend required. Deploy to Vercel, Netlify, GitHub Pages, or any static host. A `vercel.json` is included for Vercel.

## File Structure

```
index.html           — Entry point
styles.css           — All styles
script.js            — All application logic
characters.json      — Bundled character manifest
Character_Icons/     — Character portrait images
Element_Icons/       — Element icon overlays (white on transparent, 32×32)
wuwa-team-manager.json — Save file (generated by Save button)
vercel.json          — Vercel static site config
```
